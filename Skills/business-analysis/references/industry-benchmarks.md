## 1. 电商 / E-commerce

### 检测信号 Detection Signals

| 信号类型 | 匹配模式 |
|---------|---------|
| 列名关键词 | `GMV`, `SKU`, `订单`, `order`, `客单价`, `AOV`, `转化率`, `conversion`, `退货`, `return`, `购物车`, `cart`, `付款`, `payment` |
| 数据模式 | 存在「商品ID + 价格 + 数量 + 订单状态」组合；含物流/快递相关字段；有优惠券/折扣列 |
| 辅助信号 | 含平台标识（淘宝/天猫/京东/拼多多/抖音电商/Shopify）；存在店铺/商家ID |

### 核心 KPI 与健康区间

| 指标 | 定义 | 健康区间 | 优秀水平 | 警戒线 |
|------|------|---------|---------|--------|
| GMV（成交总额） | 总订单金额（含退款前） | 视规模而定 | 环比增长 >15% | 环比下降 >10% |
| 净 GMV | GMV - 退货退款金额 | GMV 的 70-85% | >85% | <65% |
| 转化率 Conversion Rate | 下单人数 / 访客数 | 2-5%（综合电商） | >5% | <1.5% |
| 客单价 AOV | 净 GMV / 订单数 | 品类差异大 | 环比提升 >5% | 持续下降 |
| 退货率 Return Rate | 退货订单 / 总订单 | 美妆 <30%；服装 <45%；3C <15% | 美妆 <20%；服装 <30% | 美妆 >40%；服装 >55% |
| 获客成本 CAC | 营销总支出 / 新客数 | 品类差异大 | CAC < 首单毛利的 50% | CAC > 首单毛利 |
| 复购率 Repurchase Rate | 二次购买客户 / 总客户 | 20-40% | >40% | <15% |
| 客户生命周期价值 LTV | 客户全周期贡献净利润 | LTV/CAC > 3 | LTV/CAC > 5 | LTV/CAC < 2 |

### 常用分析框架

- **RFM 分析**：按最近购买时间（Recency）、购买频率（Frequency）、消费金额（Monetary）分层客户
- **漏斗分析**：浏览 → 加购 → 下单 → 付款 → 收货，逐级转化率拆解
- **品类交叉分析**：SKU 级别的利润贡献矩阵（销量 x 毛利率象限图）
- **队列分析 Cohort Analysis**：按首购月份分组追踪复购行为
- **购物篮分析 Basket Analysis**：关联购买模式挖掘

### 季节性模式

| 节点 | 时间 | 影响 | 典型波动 |
|------|------|------|---------|
| 年货节 | 1月中旬 | 食品/礼品类爆发 | GMV +30-80% |
| 春节假期 | 1-2月 | 物流停摆，订单骤降 | GMV -40-60% |
| 38 女王节 | 3月初 | 美妆/服装小高峰 | 相关品类 +20-40% |
| 618 | 6月1-18日 | 全品类大促 | GMV +100-300% |
| 99 大促 | 9月初 | 中型促销节点 | GMV +30-50% |
| 双 11 | 11月1-11日 | 年度最大促销 | GMV +200-500% |
| 双 12 | 12月12日 | 中小商家大促 | GMV +50-100% |
| Black Friday | 11月下旬 | 跨境电商适用 | 跨境 GMV +100-200% |

### 典型数据 Schema

```
订单ID, 订单时间, 用户ID, SKU_ID, 商品名称, 品类, 数量, 单价,
实付金额, 优惠券金额, 订单状态(待付款/已付款/已发货/已完成/已退货),
支付方式, 收货省份, 收货城市, 物流单号, 平台来源, 流量渠道
```

## 2. SaaS / 订阅制

### 检测信号 Detection Signals

| 信号类型 | 匹配模式 |
|---------|---------|
| 列名关键词 | `MRR`, `ARR`, `churn`, `subscription`, `订阅`, `续费`, `plan`, `tier`, `seat`, `license`, `trial`, `NRR`, `expansion` |
| 数据模式 | 存在「客户 + 订阅开始/结束日期 + 月费/年费」组合；含套餐/版本等级字段 |
| 辅助信号 | 含 usage/用量数据；存在 feature flag 或权限层级；有 onboarding/activation 相关字段 |

### 核心 KPI 与健康区间

| 指标 | 定义 | 健康区间 | 优秀水平 | 警戒线 |
|------|------|---------|---------|--------|
| MRR（月度经常性收入） | 当月活跃订阅的月度收入总和 | 环比增长 >5% | 环比增长 >10% | 连续 3 月零增长 |
| ARR（年度经常性收入） | MRR x 12 | 年增长 >50%（早期） | 年增长 >100% | 年增长 <20% |
| 月流失率 Monthly Churn | 本月流失 MRR / 月初 MRR | SMB <5%；企业 <1% | SMB <3%；企业 <0.5% | SMB >8%；企业 >2% |
| 净收入留存 NRR | (月初MRR + 扩展 - 流失 - 缩减) / 月初MRR | >100% | 企业 >120%；SMB >110% | <90% |
| LTV/CAC | 客户生命周期价值 / 获客成本 | >3 | >5 | <2 |
| CAC 回收期 Payback Period | CAC / 月度毛利 | <18 个月 | <12 个月 | >24 个月 |
| 毛利率 Gross Margin | (收入 - COGS) / 收入 | >70% | >80% | <60% |
| 激活率 Activation Rate | 完成关键行为的用户 / 注册用户 | >25% | >40% | <15% |
| 试用转化率 Trial-to-Paid | 付费用户 / 试用用户 | Freemium 2-5%；Free Trial 15-30% | Freemium >5%；Trial >30% | Freemium <1%；Trial <10% |

### 常用分析框架

- **Cohort 留存分析**（最重要）：按注册月份追踪 MRR 留存曲线，找到留存拐点
- **MRR 瀑布图 MRR Waterfall**：新增 MRR / 扩展 MRR / 流失 MRR / 缩减 MRR 的月度分解
- **客户健康度评分**：基于登录频率、功能使用深度、工单数量的综合评分
- **漏斗分析**：注册 → 激活 → 付费 → 扩展，各环节转化率
- **单位经济学 Unit Economics**：LTV/CAC 比值及 CAC 回收期趋势

### 季节性模式

| 模式 | 说明 |
|------|------|
| Q1 预算效应 | 企业客户 1 月新签/续约集中（新财年预算释放） |
| Q4 冲刺效应 | 销售团队年底冲业绩，10-12 月签约量上升 |
| 夏季低谷 | 7-8 月决策周期拉长，新签速度放缓 |
| 月初/月末效应 | 月度订阅多在月初/月末集中变更 |

### 典型数据 Schema

```
客户ID, 公司名称, 行业, 公司规模(员工数), 套餐类型(Free/Basic/Pro/Enterprise),
订阅开始日期, 订阅结束日期, 月费, 年费, 席位数, 付款状态,
MRR, 上月MRR, 变更类型(新增/扩展/缩减/流失), 客户成功经理,
注册渠道, 最近登录日期, 月活跃天数, 核心功能使用次数
```

## 3. 零售 / Retail

### 检测信号 Detection Signals

| 信号类型 | 匹配模式 |
|---------|---------|
| 列名关键词 | `门店`, `store`, `POS`, `库存`, `inventory`, `坪效`, `同店`, `same-store`, `sell-through`, `动销`, `陈列`, `shelf` |
| 数据模式 | 存在「门店ID + 日期 + 销售额」组合；含商品分类层级（大类/中类/小类）；有库存量/进货量数据 |
| 辅助信号 | 含地理位置（省/市/区/商圈）字段；存在会员卡号/积分字段；有天气相关数据关联 |

### 核心 KPI 与健康区间

| 指标 | 定义 | 健康区间 | 优秀水平 | 警戒线 |
|------|------|---------|---------|--------|
| 同店增长 Same-Store Sales Growth | 同一门店同比销售增长 | >3% | >8% | <0%（连续 2 季度） |
| 坪效 Sales per sqm | 销售额 / 经营面积 | 品类差异大 | 同比提升 >10% | 同比下降 >5% |
| 库存周转天数 Inventory Days | 平均库存 / 日均销售成本 | 快消 30-45 天；服装 60-90 天 | 快消 <30 天；服装 <60 天 | 快消 >60 天；服装 >120 天 |
| 动销率 Sell-through Rate | 销售数量 / 期初库存 | >60%（季度） | >75% | <40% |
| 毛利率 Gross Margin | (售价 - 进货价) / 售价 | 超市 20-30%；服装 50-65% | 超市 >30%；服装 >65% | 超市 <18%；服装 <45% |
| 客流转化率 | 成交笔数 / 进店人次 | 20-40% | >40% | <15% |
| 连带率 Units per Transaction | 销售件数 / 交易笔数 | 服装 1.5-2.5；超市 5-10 | 服装 >2.5；超市 >10 | 服装 <1.3 |
| 缺货率 Stockout Rate | 缺货 SKU 数 / 总 SKU 数 | <5% | <2% | >10% |

### 常用分析框架

- **门店排名对比**：同区域/同面积门店的横向业绩对比，识别高/低效门店
- **品类贡献分析**：各品类的销售贡献、毛利贡献、库存占比三角对比
- **ABC 分类法**：SKU 按销售贡献分 A（前 20%）B（中间 30%）C（后 50%）三档管理
- **价格带分析**：各价格区间的销量分布，指导定价和陈列策略
- **商圈热力图分析**：基于地理位置的门店业绩可视化

### 季节性模式

| 节点 | 时间 | 影响 |
|------|------|------|
| 春节（年货采购） | 春节前 2-3 周 | 食品饮料/礼品 +100-200% |
| 春节假期 | 大年初一至初七 | 除旅游商圈外普遍下降 30-50% |
| 开学季 | 8月下旬-9月初 | 文具/服装/电子产品 +30-60% |
| 国庆黄金周 | 10月1-7日 | 旅游商圈 +50-100%；社区店平稳 |
| 换季期 | 3月/6月/9月/12月 | 服装清仓+新品上架，毛利波动大 |
| 周末效应 | 每周五-日 | 商场客流 +40-80%；便利店相对平稳 |
| 天气影响 | 极端天气 | 暴雨/极寒天气客流下降 20-40% |

### 典型数据 Schema

```
门店ID, 门店名称, 区域, 城市, 商圈类型, 面积(sqm), 日期,
SKU_ID, 商品名称, 大类, 中类, 小类, 品牌,
销售数量, 销售金额, 进货价, 售价, 折扣率, 毛利额,
库存数量, 进货数量, 退货数量,
客流量, 成交笔数, 会员占比
```

## 4. 广告投放 / Digital Advertising

### 检测信号 Detection Signals

| 信号类型 | 匹配模式 |
|---------|---------|
| 列名关键词 | `ROI`, `ROAS`, `CPM`, `CPC`, `CPA`, `CTR`, `曝光`, `impression`, `点击`, `click`, `花费`, `spend`, `投放`, `广告`, `ad`, `campaign`, `素材`, `creative` |
| 数据模式 | 存在「花费 + 曝光 + 点击 + 转化」组合；含广告计划/广告组/广告创意层级结构 |
| 辅助信号 | 含平台标识（巨量引擎/磁力引擎/腾讯广告/百度营销）；存在出价策略/定向标签字段 |

### 核心 KPI 与健康区间

| 指标 | 定义 | 健康区间 | 优秀水平 | 警戒线 |
|------|------|---------|---------|--------|
| ROI | (GMV - 广告花费) / 广告花费 | 品牌自播 >3.0；达人投放 >2.0 | 自播 >5.0；达人 >3.0 | 自播 <2.0；达人 <1.5 |
| ROAS | GMV / 广告花费 | 品牌自播 >4.0；达人 >3.0 | 自播 >6.0；达人 >4.0 | 自播 <3.0；达人 <2.5 |
| CPM（千次曝光成本） | 花费 / 曝光 x 1000 | 信息流 15-40 元；搜索 30-80 元 | 信息流 <15 元 | 信息流 >60 元 |
| CPC（单次点击成本） | 花费 / 点击数 | 0.5-3 元（信息流） | <0.5 元 | >5 元 |
| CTR（点击率） | 点击 / 曝光 | 信息流 1-3%；搜索 3-8% | 信息流 >3%；搜索 >8% | 信息流 <0.5% |
| CVR（转化率） | 转化数 / 点击数 | 电商 2-5%；线索 5-15% | 电商 >5% | 电商 <1% |
| CPA（单次获客成本） | 花费 / 转化数 | 视行业而定 | 环比下降 >10% | 环比上升 >20% |
| 投放费比 Spend/GMV Ratio | 广告花费 / GMV | 品牌自播 <18%；全渠道 <25% | <12% | >30% |

#### 分平台基准

| 平台 | CPM 参考 | CTR 参考 | CVR 参考 | 特殊指标 |
|------|---------|---------|---------|---------|
| 抖音/巨量引擎 | 20-50 元 | 1-4% | 2-6% | GPM（千次曝光成交额）>800 元（优秀） |
| 小红书/聚光 | 30-80 元 | 2-5% | 1-3% | 笔记互动成本 <5 元（优秀） |
| 微信朋友圈 | 50-120 元 | 0.8-2% | 1-4% | 关注成本 <10 元（公众号） |
| 腾讯广告（非朋友圈） | 15-40 元 | 1-3% | 2-5% | 表单成本视行业 |
| 百度搜索 | 30-100 元 | 3-8% | 5-15% | 关键词质量度 >7（优秀） |
| 快手/磁力引擎 | 10-30 元 | 1-3% | 2-5% | 下沉市场 CPC 更低 |

### 常用分析框架

- **素材效率分析**：按创意素材维度统计 CTR/CVR/ROI，识别最优素材类型
- **投放漏斗分析**：曝光 → 点击 → 加购/留资 → 成交，逐级转化效率
- **人群包对比**：不同定向人群的效率对比（兴趣/行为/再营销/相似人群）
- **时段分析**：24 小时内各时段的 CPC/CVR 变化，指导分时出价
- **边际 ROI 分析**：每增加 1000 元预算，ROI 如何衰减，找到最优预算规模

### 季节性模式

| 模式 | 说明 |
|------|------|
| 大促前 CPM 上涨 | 618/双 11 前 2 周，CPM 普遍上涨 30-80% |
| 春节流量低谷 | 春节期间平台活跃度下降，但 CPM 也低，可能出现机会 |
| 开年预算释放 | Q1 品牌预算释放，1-3 月竞争加剧 |
| Q4 冲量期 | 10-12 月，各平台流量政策放宽，CPM 波动大 |
| 周中/周末差异 | 信息流 CTR 周末略高；B2B 类搜索广告工作日更优 |

### 典型数据 Schema

```
日期, 广告计划ID, 广告组ID, 素材ID, 平台, 投放目标(成交/线索/关注),
花费, 曝光数, 点击数, CTR, CPC, 转化数, CVR, CPA,
GMV(成交金额), ROI, ROAS,
定向人群, 版位, 出价方式, 素材类型(图片/视频/直播)
```

## 5. 内容 / Content & Media

### 检测信号 Detection Signals

| 信号类型 | 匹配模式 |
|---------|---------|
| 列名关键词 | `阅读量`, `read`, `播放量`, `view`, `点赞`, `like`, `评论`, `comment`, `转发`, `share`, `粉丝`, `follower`, `互动率`, `engagement`, `完播率`, `watch_time` |
| 数据模式 | 存在「内容ID + 发布时间 + 互动指标组」组合；含内容类型/标签字段 |
| 辅助信号 | 含平台标识（抖音/B站/小红书/公众号/视频号）；存在作者/账号相关字段 |

### 核心 KPI 与健康区间

| 指标 | 定义 | 健康区间 | 优秀水平 | 警戒线 |
|------|------|---------|---------|--------|
| 互动率 Engagement Rate | (点赞+评论+转发+收藏) / 曝光 | 小红书 3-8%；抖音 2-5%；公众号 1-3% | 小红书 >10%；抖音 >8% | <1% |
| 完播率 Completion Rate | 完整播放次数 / 播放次数 | 短视频(15s) >40%；中视频(3min) >20% | 短视频 >60%；中视频 >35% | 短视频 <25% |
| 涨粉率 Follower Growth | 新增粉丝 / 曝光 | 0.5-2% | >3% | 持续 <0.3% |
| 内容 ROI | 内容带来的 GMV / 内容制作成本 | >5（自产内容）；>2（达人合作） | 自产 >10 | <2 |
| 爆款率 Viral Rate | 播放量 >10 万的内容占比 | 5-10% | >15% | <3% |
| 粉丝粘性 | 粉丝互动 / 总互动 | >30% | >50% | <15% |

#### 分平台基准

| 平台 | 核心指标 | 良好基准 | 说明 |
|------|---------|---------|------|
| 抖音 | 完播率、互动率、转粉率 | 完播率 >35%（短视频），互动率 >3% | 算法权重：完播率 > 互动率 > 转粉率 |
| B 站 | 播放完成度、一键三连率、弹幕密度 | 一键三连率 >5%，弹幕 >50 条/万播放 | 中长视频为主，完播率标准不同 |
| 小红书 | 收藏率、互动率、搜索排名 | 收藏率 >5%，互动率 >5% | 收藏权重极高，影响搜索排名 |
| 公众号 | 打开率、分享率、在看率 | 打开率 >3%（服务号 >8%），分享率 >2% | 打开率逐年下降趋势 |
| 视频号 | 完播率、社交推荐率 | 完播率 >30%，来自好友推荐 >20% | 社交推荐是核心分发机制 |

### 常用分析框架

- **内容标签效率矩阵**：按话题/格式/时长维度统计各类内容的平均表现
- **发布时间分析**：不同发布时段的曝光量和互动率差异
- **爆款要素拆解**：对 Top 10% 内容做归因分析（标题/封面/时长/话题/发布时间）
- **粉丝画像对比**：高互动粉丝 vs 沉默粉丝的特征差异
- **内容生命周期分析**：内容从发布到流量衰减的时间曲线

### 季节性模式

| 模式 | 说明 |
|------|------|
| 春节内容消费高峰 | 除夕至初七，短视频/直播观看时长 +30-50% |
| 假期后开工低谷 | 假期结束后 1 周，工作日内容消费恢复正常 |
| 高考/考研季 | 教育类内容 6 月/12 月流量高峰 |
| 年末盘点 | 12 月各类「年度总结」内容关注度高 |
| 社会热点脉冲 | 突发事件/热搜会在 24-48 小时内带来巨大流量波动 |

### 典型数据 Schema

```
内容ID, 发布时间, 平台, 账号ID, 账号名称, 内容类型(图文/短视频/中视频/直播),
标题, 标签/话题, 时长(秒),
播放量/阅读量, 点赞数, 评论数, 转发数, 收藏数, 新增粉丝数,
互动率, 完播率, 平均观看时长,
来源分布(推荐/搜索/关注/社交分享)
```

## 6. 餐饮 / Food & Beverage

### 检测信号 Detection Signals

| 信号类型 | 匹配模式 |
|---------|---------|
| 列名关键词 | `翻台率`, `table_turnover`, `客单价`, `check`, `食材成本`, `food_cost`, `人力成本`, `labor`, `外卖`, `delivery`, `堂食`, `dine-in`, `菜品`, `menu`, `桌号`, `table` |
| 数据模式 | 存在「门店 + 日期 + 营业额 + 桌/单量」组合；含菜品明细和食材成本数据 |
| 辅助信号 | 含外卖平台（美团/饿了么）数据；存在时段（早/中/晚/夜宵）划分；有原材料采购记录 |

### 核心 KPI 与健康区间

| 指标 | 定义 | 健康区间 | 优秀水平 | 警戒线 |
|------|------|---------|---------|--------|
| 翻台率 Table Turnover | 日接待桌数 / 总桌数 | 正餐 2-3 次；快餐 5-8 次 | 正餐 >3.5；快餐 >10 | 正餐 <1.5；快餐 <4 |
| 客单价 Average Check | 营业额 / 交易笔数 | 视定位而定 | 环比提升且客流不降 | 持续下降 >10% |
| 食材成本率 Food Cost Ratio | 食材成本 / 营业额 | 28-35% | <28%（品质不降） | >38% |
| 人力成本率 Labor Cost Ratio | 人力成本 / 营业额 | 25-30% | <22% | >35% |
| 租金占比 Rent Ratio | 租金 / 营业额 | 8-15% | <8% | >18% |
| 四费合计（食材+人力+租金+水电） | 四项主要成本之和 / 营业额 | 75-85% | <75% | >88% |
| 外卖占比 Delivery Ratio | 外卖营业额 / 总营业额 | 20-40% | 取决于策略 | 堂食不足导致 >60% |
| 菜品毛利率 Dish Margin | (售价 - 食材成本) / 售价 | 60-70% | >70% | <55% |
| 退菜率 Return Rate | 退菜次数 / 总下单菜品数 | <2% | <1% | >5% |

### 常用分析框架

- **菜品 ABC 分析**：按销量和毛利率将菜品分为明星/金牛/瘦狗，指导菜单优化
- **时段经营分析**：早/午/晚/夜宵各时段的营收贡献和坪效
- **堂食 vs 外卖对比**：两个渠道的客单价、毛利率、订单结构差异
- **食材波动追踪**：核心食材价格变动对毛利率的影响测算
- **人效分析**：营业额 / 员工数，对比不同班次和岗位的效率

### 季节性模式

| 模式 | 说明 |
|------|------|
| 春节爆发 | 年夜饭/聚餐，正餐类营收 +50-100%（需提前备货） |
| 节后低谷 | 正月初七至元宵后 1 周，到店客流恢复期 |
| 夏季冷饮高峰 | 饮品/甜品/冰品类 6-8 月销量 +50-100% |
| 冬季火锅/热饮高峰 | 11-2 月火锅/热饮类营收显著上升 |
| 周五-周日高峰 | 正餐翻台率周末 vs 工作日高 30-60% |
| 恶劣天气效应 | 暴雨/极端天气：堂食下降 30-50%，外卖上升 20-40% |

### 典型数据 Schema

```
门店ID, 门店名称, 日期, 时段(早/午/晚/夜宵), 渠道(堂食/外卖/自提),
订单ID, 桌号, 人数, 菜品ID, 菜品名称, 菜品分类, 数量, 售价, 食材成本,
订单金额, 折扣金额, 实收金额, 支付方式,
外卖平台, 平台佣金率,
当日员工数, 当日人力成本, 食材采购总额
```

## 7. 制造业 / Manufacturing

### 检测信号 Detection Signals

| 信号类型 | 匹配模式 |
|---------|---------|
| 列名关键词 | `OEE`, `良品率`, `yield`, `不良率`, `defect`, `产能`, `capacity`, `工单`, `work_order`, `产线`, `production_line`, `BOM`, `物料`, `material`, `设备`, `machine` |
| 数据模式 | 存在「工单/批次 + 计划产量 + 实际产量 + 不良数量」组合；含设备/产线编号 |
| 辅助信号 | 含供应商/物料编码字段；存在质检相关数据；有设备运行时间/停机记录 |

### 核心 KPI 与健康区间

| 指标 | 定义 | 健康区间 | 优秀水平 | 警戒线 |
|------|------|---------|---------|--------|
| OEE（设备综合效率） | 可用率 x 性能率 x 良品率 | 60-75% | >85%（世界级） | <55% |
| 良品率 Yield Rate | 良品数 / 总产出 | >95% | >99%（6 Sigma 水平） | <90% |
| 不良率 Defect Rate | 不良品数 / 总产出 | <5% | <1% | >10% |
| 产能利用率 Capacity Utilization | 实际产量 / 额定产能 | 75-90% | 85-92%（过高反而风险大） | <65% |
| 库存周转天数 Inventory Days | 原材料+在制品+成品库存 / 日均消耗 | 原材料 15-30 天；成品 10-20 天 | 精益模式 <10 天 | >45 天 |
| 准时交付率 On-time Delivery | 按时交付订单 / 总订单 | >90% | >98% | <85% |
| 设备停机率 Downtime Rate | 计划外停机时间 / 总可用时间 | <10% | <5% | >15% |
| 人均产出 Output per Worker | 产量 / 直接人工数 | 视行业而定 | 环比提升 >5% | 环比下降 >10% |
| 废品率 Scrap Rate | 报废物料成本 / 总物料成本 | <3% | <1% | >5% |
| 订单达成率 | 实际完成量 / 计划量 | >95% | >99% | <90% |

### 常用分析框架

- **OEE 三因子分解**：将 OEE 拆为可用率（Availability）x 性能率（Performance）x 良品率（Quality），精准定位瓶颈
- **帕累托分析（不良原因）**：按不良原因分类排序，找到占 80% 不良的前 20% 原因
- **SPC 统计过程控制**：用控制图监控关键工序参数，判断过程是否受控
- **产线平衡分析**：各工序节拍时间对比，识别瓶颈工序
- **供应商绩效矩阵**：按交期准时率、来料合格率、价格竞争力综合评估供应商

### 季节性模式

| 模式 | 说明 |
|------|------|
| 春节停产期 | 1-2 月产能大幅下降，节后复工需要产能爬坡期（通常 1-2 周） |
| 订单季节性 | 下游行业决定：消费电子 Q3-Q4 旺季备货；服装 Q1/Q3 换季备货 |
| 原材料价格波动 | 大宗商品价格周期性波动影响物料成本 |
| 设备检修周期 | 通常安排在淡季集中大修，影响产能 |
| 年底冲产量 | 12 月为完成年度目标，产能利用率可能异常偏高 |

### 典型数据 Schema

```
工单ID, 产品编码, 产品名称, 产线ID, 班次(早/中/晚),
日期, 计划产量, 实际产量, 良品数, 不良数,
不良分类(外观/尺寸/功能/其他), 不良原因,
设备ID, 设备名称, 运行时间(分钟), 停机时间(分钟), 停机原因,
物料编码, 物料名称, BOM用量, 实际用量, 物料成本,
供应商ID, 来料批次, 来料检验结果,
操作员ID, 工序名称, 工序节拍时间(秒)
```

## 使用指南

### Probe 阶段调用逻辑

```
1. 扫描数据列名 → 与各行业「检测信号」做模糊匹配
2. 命中率最高的行业 → 加载该行业基准
3. 若命中多个行业（如电商+广告投放）→ 同时加载，交叉分析
4. 若无明确命中 → 跳过行业基准，使用通用分析框架
```

### 基准引用规范

在分析报告中引用行业基准时，采用以下格式：

```
[发现] 当前退货率为 52%
[基准] 服装行业健康区间为 <45%，当前已超出警戒线
[建议] 需深入分析退货原因分布（尺码/质量/预期不符），优先解决占比最高的退货原因
```

### 基准数据局限性说明

- 以上基准主要反映**中国市场**的行业均值，海外市场需调整
- 数据来源为行业公开报告和从业经验汇总，非精确统计值
- 不同细分赛道（如高端 vs 大众）的基准差异可能很大，需结合具体上下文判断
- 基准数据应作为**参考锚点**而非绝对标准，核心价值在于发现异常值和变化趋势
